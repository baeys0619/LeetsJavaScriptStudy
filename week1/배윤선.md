### Number

- 정수, 실수, 음수 등 모든 숫자가 number type에 포함됨
- 기본적인 모든 사칙연산을 지원함
- 추가적으로 %(모듈러 연산)도 지원함
- Infinity(양의 무한대),-Infinity(음의 무한대), NaN도 number type

### String

- 문자열을 의미함
- 문자열은 무조건 쌍따옴표나 작은따옴표로 묶어줘야 함
- 덧셈 연산자를 지원함
    
    ```bash
    let Name = "Anna";
    let Location = "Korea";
    let Introduction = Name + Location;
    
    //AnnaKorea
    ```
    
- **템플릿 리터럴 문법** : 백틱기호(`)를 사용할 수도 있음 → 변수${}와 문자를 섞어서 사용할 수 있게 해줌

```bash
...
let introduction = `${Name}는 ${Location}에 거주합니다`;

//Anna는 Korea에 거주합니다
```

### Boolean

- True나 False값만을 가진다
- 현재 상태를 의미하는데 주로 사용

### Null

예시) `let empty = null;` //empty엔 아무것도 없음… 

### Undefined

- 변수를 선언하고 아무것도 할당하지 X → undefined 할당

## 묵시적 형 변환

→ 개발자가 직접 설정하지 않아도 알아서 JS엔진이 형 변환하는 것을 말함 

```bash
let num = 10;
let str = "20";

console.log(result);

//1020
```

- JS엔진에서 자체적으로 num을 String로 변환함
- 단, 모든 경우에 묵시적 형 변환이 먹히는 것은 아님

## 명시적 형 변환

→ 개발자가 직접 내장함수 등을 이용해 형 변환을 일으킴


📌parseInt() 주의점 : 최대한 근데 그래도 숫자가 앞으로 와야 얘가 에러가 안 나고 인식을 잘함

<aside>

🧐왜 ==가 아니라 ===를 쓰지?

== : 값 자체로만 비교하지 타입까지 비교하진 않음

=== : 값 뿐만 아니라 타입(자료형)까지 비교를 하게 됨 

```jsx
let comp1 = 1 == '1'; //true
```

## null 병합 연산자

기호 : `??`

- 존재하는 값을 추려내는 기능을 함
- null, undefined가 아닌 값을 찾아냄

```jsx
let var1;
let var2 = 10;
let var3 = 20;

let var4 = var1 ?? var2;
console.log(var4); //10

//이유 : var1은 초기화가 안됐으니까 undefined고 var2는 10이므로 
//undefined가 아닌 var2의 값을 출력함
```

- 근데 만약 null병합연산자로 묶인 애들이 둘 다 null이나 undefined값이 아니면 그냥 앞에 있는 변수의 값을 출력함

## TypeOf 연산자

- 값의 타입을 문자열로 반환하는 기능을 함
- 변수의 타입이 고정되어 있지 않음, 근데 만약 지금 저장된 변수의 타입이 궁금하다면? typeof연산자를 쓰면 되는 거임

```jsx
let var7 = 1;
var7 = "hello";

let t1 = typeof var7;
console.log(t1); //String
```

## 삼항연산자

- 항(약간 덩어리같은 느낌)을 3개 사용하는 연산자라서 **삼항연산자**임
- 조건식을 이용해서 참/거짓일 때의 값을 다르게 반환
- 형식 : `조건문 ? 참일 대의 결과 : 거짓일 때의 결과`

예시) 변수 res에 var8의 값이 짝수면 ‘짝’, 홀수면 ‘홀’을 출력하도록 함 

```jsx
let res = var8%2 === 0 ? "짝" : "홀";
```
